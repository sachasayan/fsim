## 2024-05-24 - Canvas Opacity and Dashboard Navigation Landmarks
**Learning:** `<canvas>` elements are completely opaque to screen readers by default. Wrapping them in accessible `div` containers is not enough; the `role="img"` and `aria-label` must be applied directly to the canvas element itself (both for standard 2D canvases like the minimap and dynamic WebGL contexts like the main Three.js view). Additionally, complex dashboard layouts with distinct sub-screens (like ND and EICAS in flight simulators) need `role="group"` along with descriptive `aria-label`s on their containers to provide meaningful navigation landmarks for assistive technologies.
**Action:** Always apply `role="img"` and `aria-label` directly to generated `<canvas>` elements, and use grouped landmarks for complex multi-panel dashboard UIs.
