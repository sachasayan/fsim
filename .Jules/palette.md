## 2024-05-24 - Semantic landmarks for flight dashboard and opaque canvases
**Learning:** For complex dashboard interfaces with multiple screen regions (e.g., flight simulator panels like ND or EICAS), `role="group"` combined with descriptive `aria-label`s provide essential semantic navigation landmarks for assistive technologies. Furthermore, when adding accessibility to WebGL/Three.js applications, `role="img"` and a descriptive `aria-label` should be applied directly to the `<canvas>` element itself because canvas elements are inherently opaque to screen readers, and wrapping container `div`s lack the necessary semantic meaning to resolve this out-of-the-box.
**Action:** Always verify that complex UI regions have semantic grouping (`role="group"`) and that any `<canvas>` element presenting meaningful visual information has `role="img"` and an `aria-label`.
