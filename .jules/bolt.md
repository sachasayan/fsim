## 2024-03-08 - [Optimizing Perlin/Fractal Noise using inline gradients]
**Learning:** Math.floor and bitwise operations inside the hot-loop of the noise function create a slight bottleneck, but a major optimization opportunity exists in skipping array lookup functions or branch statements for gradients. Inlining pre-calculated Float32Arrays for `gradX`, `gradY`, and `gradZ` and fetching from them using hash indices drastically improves the `Noise.noise` runtime performance. Removing bitwise ternary logic and nested calls from the innermost `lerp` combinations leads to massive speedups.
**Action:** When working on heavily utilized procedural generation or noise functions, pre-calculate gradients and inline lerp operations directly in the function body rather than abstracting them into method calls. Ensure arrays are pre-allocated and strongly typed (`Float32Array`).
