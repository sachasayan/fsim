## 2024-05-24 - Inline Math & Hoist TypedArrays in High-Frequency Noise Functions
**Learning:** In V8, high-frequency mathematical functions (like `noise.noise(x,y,z)` called 100,000+ times) suffer significant overhead from repeated property accesses (`this.fade`, `this.lerp`, `this.grad`, `this.permutation`). Inlining these simple math operations directly and extracting the typed array to a module-scoped variable (`const P = new Uint8Array(512)`) avoids context lookups entirely, speeding up execution significantly.
**Action:** When working on math-heavy hot loops (especially terrain, physics, or particle systems), inline core math functions explicitly, expand logical operations, and extract static configuration objects out to module scope rather than as object properties.
