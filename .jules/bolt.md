## 2025-04-04 - Inlining 3D Perlin Noise gradients
**Learning:** For extremely hot loops like \`Noise.fractal\`, hoisting variables like arrays into the global scope and heavily manually inlining variables (such as nested math functions \`grad\`, \`fade\`, \`lerp\`) and replacing complex control flow (like function calls in nested conditionals) with direct calculations using ternary operators results in massive V8 optimization. However, replacing math functions directly with an array map loop actually negatively impacted performance slightly compared to unrolled branch-based evaluations via ternary operators. Removing the \`permutation\` property from the \`NoiseOpt\` namespace entirely, and relying on closure lookup rather than \`this.permutation\` lookup was also a large win.
**Action:** For heavy micro-optimizations of math routines on the frontend in V8, inline aggressively. Unrolling math code directly into the calling function rather than modular functions saves instruction pointer jumps, and prefer straight closure lookups over object properties to bypass getter overhead. Avoid converting ternary branches to array lookups in deep hot paths because index bounds-checking is more expensive than branch-prediction.
