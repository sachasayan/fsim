## 2024-05-19 - Extreme V8 Noise Optimization
**Learning:** In extreme performance math loops (like per-vertex noise generation), flat math with heavy inlining performs significantly better than clean abstraction in V8. Moving static arrays to module-scoped constants instead of accessing them via `this.property`, heavily inlining nested gradients, and unrolling loops to avoid conditionals yields a ~35% speedup in procedural generation hot loops.
**Action:** When optimizing object methods called frequently in hot loops (e.g., `Noise.noise`), hoist frequently accessed instance properties like static arrays out of the object into module-scoped constants to eliminate `this.` context property lookup overhead. Flatten the algorithm into explicit primitives by heavily inlining operations.
